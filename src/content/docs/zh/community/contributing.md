感谢您对为 C2PA Wiki 做出贡献的兴趣！本指南将帮助您入门。

## 贡献方式

### 1. 改进文档

- 修复拼写错误或澄清说明
- 添加示例和教程
- 更新过时信息
- 将内容翻译成新语言

### 2. 报告问题

发现问题？请告诉我们：
- [报告文档问题](https://github.com/paulortiz199928/c2pa-wiki/issues)
- [报告内容问题](https://github.com/paulortiz199928/awesome-c2pa/issues)
- [报告翻译错误](https://github.com/paulortiz199928/awesome-c2pa/issues/new?template=translation.md)

### 3. 添加资源

帮助扩展 awesome-c2pa 列表：
- 新工具和库
- 教程和文章
- 用例和演示
- 实现示例

### 4. 改进翻译

帮助让 C2PA 在全球范围内可访问：
- 审核 AI 生成的翻译
- 将文档翻译成新语言
- 更新现有翻译

## 入门

### 对于 awesome-c2pa 仓库

这是所有内容的源头：

1. Fork [awesome-c2pa](https://github.com/paulortiz199928/awesome-c2pa) 仓库
2. 创建功能分支：`git checkout -b feature/my-contribution`
3. 进行更改
4. 提交拉取请求

### 对于 c2pa-wiki 仓库

用于基础设施和 UI 改进：

1. Fork [c2pa-wiki](https://github.com/paulortiz199928/c2pa-wiki) 仓库
2. 安装依赖：`npm install`
3. 运行开发服务器：`npm run dev`
4. 进行更改并在本地测试
5. 提交拉取请求

**注意**：不要直接在 c2pa-wiki 中编辑同步的内容 - 它会被覆盖。请在 awesome-c2pa 中编辑源内容。

## 内容指南

### 质量标准

贡献应该：
- **准确**：经过事实核查且技术正确
- **相关**：与 C2PA 实现或理解相关
- **写得好**：清晰、简洁且格式正确
- **最新**：最近且积极维护

### Markdown 风格

使用 GitHub 风格的 markdown：
- 使用标题层次结构（h1、h2、h3）
- 添加带有语言标签的代码块
- 包含描述性链接文本
- 使用表格显示结构化数据

## 翻译指南

请参阅我们的 [翻译指南](/c2pa-wiki/community/translations/) 了解详细信息。

## 行为准则

保持尊重和专业：
- 欢迎新手
- 接受建设性批评
- 关注对社区最有利的事情
- 对他人表现出同理心
